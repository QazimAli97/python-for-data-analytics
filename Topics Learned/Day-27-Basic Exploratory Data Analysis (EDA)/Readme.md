# 🐼 Day 27 – Basic Exploratory Data Analysis (EDA) using Pandas

## 📌 Introduction
Day 27 focuses on **Exploratory Data Analysis (EDA)** using Pandas.

EDA is the process of exploring datasets to understand their structure, patterns, and key statistics before performing deeper analysis.

It is one of the first steps in any data analysis project.

---

## 🎯 Objective
The goal of Day 27 is to:

- Understand dataset structure
- Explore dataset shape and columns
- Generate statistical summaries
- Identify data distribution patterns
- Extract basic insights

---

## 🧠 Why EDA Matters in Data Analytics

In real-world analytics workflows:

- Analysts explore datasets before building reports
- Data patterns are identified early
- Data quality issues are detected
- Key metrics are discovered

EDA helps analysts understand what the data is telling.

---

## 🔎 Concepts Covered

- `.shape`
- `.columns`
- `.info()`
- `.describe()`
- `value_counts()`
- Basic statistical calculations

---

## 🧪 Script Summary

The script demonstrates:

- Creating a sample sales dataset
- Calculating revenue
- Exploring dataset structure
- Generating statistical summaries
- Counting product frequency
- Calculating average revenue

This simulates the first step of data exploration in analytics.

---

## 💡 Key Learning

EDA helps analysts quickly understand the dataset before deeper analysis.

Understanding exploratory analysis prepares the foundation for:

- Data visualization
- Trend analysis
- Predictive modeling
- Business insights generation

Exploring data leads to better questions and better insights.
